# Security Policy

## Supported Versions

`qwykz` is an actively maintained open-source developer tool. Security patches, vulnerability hotfixes, and architectural updates are strictly applied to the latest major release branch.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

**Please do not report security flaws or sensitive vulnerabilities through public GitHub issues.**

If you discover a potential security risk in `qwykz` (such as insecure template stubs, unmasked secret generation, or command injection risks), please report it responsibly using the repository's secure channel:

1. **GitHub Private Vulnerability Reporting (Preferred):** Navigate to the [Security tab of this repository](https://github.com/MarcZxc1/qwykz/security/advisories) and click **"Report a vulnerability"**. This opens a secure, encrypted draft where the maintainer and the reporter can privately audit the code and prepare a patch.

### What to expect:
* **Acknowledgment:** We aim to triage and acknowledge all submitted vulnerability reports within **48 hours**.
* **Patch Timeline:** If the vulnerability is confirmed, we will provide an estimated timeline for pushing a hotfix release (e.g., `v1.0.7`). 
* **Recognition:** Security researchers and developers who practice responsible disclosure will be formally credited in the official GitHub Security Advisory and release notes once the fix is public.
